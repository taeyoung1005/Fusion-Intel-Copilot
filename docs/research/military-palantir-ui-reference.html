Codex harness architecture

Python 고정이 아니라, 서버 뒤쪽 실행 어댑터를 선택한다

공식 문서 기준으로 Codex는 SDK와 CLI 비대화 실행 모두 가능하다. 이 프로젝트는 이미 TypeScript/Vite 서버가 있으므로 프론트 계약은 `/api/codex-agent`로 유지하고, 뒤쪽 worker만 교체 가능한 구조가 가장 안전하다.

방식 근거 장점 단점 D4D 추천
현재 HTTP provider adapter 서버 내부 `/api/codex-agent`가 외부 worker endpoint로 POST 언어 독립, 데모 안정성, mock 쉬움 실제 Codex thread/session 기능은 worker가 직접 구현해야 함 유지. 프론트/서버 계약으로 남긴다.
TypeScript Codex SDK OpenAI Codex SDK 문서가 `@openai/codex-sdk` 사용 예시를 제공한다. 현재 Node 서버와 언어가 맞음, thread continuation 구현이 자연스러움 서버 런타임에서 Codex 인증/샌드박스 정책 설계 필요 1순위 후보. 백엔드가 TS로 남는다면 이쪽.
Python Codex SDK 같은 공식 SDK 문서가 `openai_codex`와 `Sandbox` 예시를 제공한다. CV/YOLO/데이터 파이프라인이 Python이면 붙이기 쉬움 현재 앱에 Python worker 운영 복잡도가 추가됨 CV 분석 worker가 Python 중심으로 커질 때 채택.
`codex exec` 비대화 CLI OpenAI non-interactive mode와 CLI reference가 스크립트/CI용 `codex exec`를 설명한다. 프로토타입 빠름, shell script로 바로 연결 가능 장기 session, streaming, 중간 상태 UI에는 SDK보다 거칠 수 있음 해커톤 데모의 빠른 proof에는 좋음. 제품형은 SDK 선호.

Design principles

군용 느낌을 내되, 무기 조작 UI처럼 보이면 안 된다

D4D 데모의 설득력은 “사람 대신 판단”이 아니라 “센서·증거·누락 맥락을 묶어 사람이 더 빨리 판단하게 한다”에 있다. 그래서 화면도 공격적 액션보다 관제·증거·승인 흐름이 중심이어야 한다.

1. Common Operational Picture

지도/카메라/이벤트를 한 화면에서 상호 참조한다. 카메라별 판단보다 “시설/구역 단위 상황”이 주인공이다.

2. Evidence-first agent

Codex는 결론부터 말하지 않는다. 관측, 근거, 반례, 누락 데이터, 다음 확인 질문 순서로 표시한다.

3. Human confirmation gate

수하, 보고, 출동 같은 비용 큰 행동은 버튼 하나로 처리하지 않는다. 절차 단계와 승인 흔적을 남긴다.

4. Dense but legible

큰 카드와 과한 여백 대신, 작은 패널·타임라인·상태 rail로 밀도 있게 구성한다. 단 CJK 줄바꿈은 절대 깨지면 안 된다.

5. Sensor fusion over single feed

1개 CCTV가 아니라 카메라 그룹, blind spot handoff, 반복 출현, 거리 변화가 한 사건으로 합쳐지는 느낌을 만든다.

6. Audit trail as product surface

일일보고/주간보고는 부가 기능이 아니라 핵심 산출물이다. 모든 판단은 citation과 operator action으로 되돌아가야 한다.

Recommendation

최종 추천: Palantir Blueprint + ArcGIS COP + Elastic SOC + Codex SDK worker

D4D 데모를 가장 설득력 있게 만드는 조합은 “지도형 COP, 다중 CCTV sensor fusion, 증거 기반 agent reasoning, 사람 승인 gate, 자동 일일/주간 보고”다.

바로 적용할 UI 변경

  • 현재 3열 dashboard에 중앙 “facility COP / camera graph” 패널을 추가한다.
  • 오른쪽 Codex 패널을 “판단 → 반례 → 누락 맥락 → 권고 절차” 순서로 재구성한다.
  • CCTV wall은 단순 grid가 아니라 camera group, blind spot handoff, 반복 출현 rail을 표시한다.
  • 일일보고/주간보고는 별도 report workspace로 빼고 citation table을 고정 노출한다.

바로 적용하지 말 것

  • 과한 neon, crosshair, target lock, weapon HUD처럼 보이는 장식.
  • 빨간색을 기본 상태로 쓰는 화면. 실제 경보 단계가 무뎌진다.
  • Palantir/Anduril 로고나 제품 화면을 직접 복제하는 것.
  • Codex SDK를 프론트 브라우저에서 직접 호출하는 구조. 서버 worker 뒤에 둔다.

Codex worker 선택 기준

  • 현재 TS 서버 유지: TypeScript Codex SDK를 worker로 연결.
  • CV/YOLO Python pipeline이 주도: Python Codex SDK worker 도입.
  • 해커톤에서 빠른 데모: `codex exec`를 queue worker로 감싸기.
  • 제품형 확장: HTTP adapter contract 유지, worker는 SDK 기반으로 교체.

다음 구현 단위

  • `DESIGN.md`를 v2 토큰으로 갱신한다.
  • facility COP 컴포넌트와 camera group memory panel을 추가한다.
  • `/api/codex-agent` 뒤쪽에 Codex worker strategy enum을 추가한다.
  • Playwright visual QA를 390, 768, 1440 viewport로 다시 돌린다.
