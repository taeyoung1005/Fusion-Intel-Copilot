# EVIDENCE CLIPS 제거 + EVENT TIMELINE 상세 표시 + 실시간 탐지 알림 팝업

## Context

D4D COP 대시보드는 지금 두 개의 겹치는 증거 표시 영역을 갖고 있다: `EVENT TIMELINE`(시간축 위 이벤트 블록)과 `EVIDENCE CLIPS`(같은 데이터를 카드 스트립으로 다시 보여주는 섹션). 사용자는 이 중복을 없애고, 대신 실시간으로 뭔가 탐지되면 화면에 CCTV 영상 팝업이 뜨는 더 즉각적인 알림 방식을 원한다. 목표는 "타임라인은 훑어보는 용도, 상세는 클릭/호버로 온디맨드, 급한 건 팝업으로 바로 눈에 띄게"로 세 가지 UI 요소의 역할을 명확히 나누는 것이다.

이 스펙은 브레인스토밍 로드맵(A → B → D → E)의 **A(UI 재구성)** 단계만 다룬다. B(DETR 속성 추출), D(크로스 카메라 재식별), E(Codex 시간 윈도우 정책)는 각각 별도 스펙으로 이어서 진행한다.

## 1. EVIDENCE CLIPS 섹션 제거

`src/cop/EvidenceClips.tsx`(카드 스트립 컴포넌트)를 삭제하고 `CopDashboard.tsx`에서 렌더링을 제거한다. `src/cop/ClipPlayer.tsx`(캡처 프레임 + 스캔 오버레이 + 신뢰도를 보여주는 재생 모달)는 그대로 유지하고 재사용 대상을 EVIDENCE CLIPS의 "재생" 버튼에서 EVENT TIMELINE의 클릭 핸들러로 옮긴다.

## 2. EVENT TIMELINE 인터랙션

`src/cop/EventTimeline.tsx`의 각 타임라인 블록(`.cop-track-block`)에:
- **호버**: 간단한 툴팁(라벨, 시각, 신뢰도)을 블록 근처에 표시. 새로 작은 툴팁 컴포넌트를 추가한다.
- **클릭**: 기존 `selectTimelineEvent` 핸들러(이미 `selectedClipId`를 설정함)가 그대로 유지되고, 여기에 더해 `ClipPlayer` 모달을 오픈하는 상태(`isClipPlayerOpen` 같은)를 추가해 클릭 시 모달이 뜨도록 한다.

## 3. 실시간 탐지 알림 팝업

### 3.1 트리거 감지
새 컴포넌트/훅(예: `useRealtimeAlerts`)이 `CopDashboard`의 `visionEvidence` state 변화를 지켜본다. 매 렌더에서 새로 추가된 클립(이전 렌더에는 없던 `id`)을 찾아 다음 조건으로 필터링한다:
- **CARLA 카메라 탐지인지 판별**: `clip.id.startsWith("ev-carla-vision-")`로 판별한다. 웹캠 테스트 패널(`RealTimeVisionPanel`)의 클립은 `ev-vision-...` 접두사를 쓰므로 자동으로 제외된다 — `clip.camera` 필드만으로는 구분이 안 되는데, 우측 웹캠 패널의 `cameraLabel` prop이 첫 번째 CARLA 카메라의 실제 id를 재사용하기 때문이다(`CopDashboard.tsx`의 `liveCameraLabel`).

### 3.2 "같은 사건 진행 중" 판단
카메라 id별 마지막 탐지 시각을 `Map<cameraId, number>`(ref)로 추적한다:
- 새 클립 도착 시, 그 카메라의 마지막 기록 시각과 지금 시각의 차이가 임계값(초기값 **8초** — 실제 사용해보면서 조정 가능하도록 상수 하나로 분리) 이상이면 **새 팝업**을 연다.
- 임계값 이내면 팝업을 새로 열지 않는다(사용자가 이전 팝업을 이미 닫았어도 다시 열지 않음 — "같은 사건"으로 취급).
- 마지막 기록 시각은 매 클립마다 갱신한다(팝업을 새로 열든 안 열든).

### 3.3 팝업 상태/렌더링
- 상태: `activeAlerts: readonly { id: string; cameraId: string; clip: EvidenceClip; autoClose: boolean; autoCloseMs: number }[]`
- 각 팝업은:
  - 해당 카메라의 라이브 MJPEG 스트림(`carlaCameraStreamSrc(cameraId)`, 기존 `CarlaCctvWall`과 동일한 방식)을 `<img>`로 표시
  - 탐지 라벨/신뢰도 텍스트
  - 우측 상단 X 버튼(수동 닫기)
  - 우측 상단 톱니바퀴 아이콘 → 클릭 시 작은 인라인 패널: "자동 닫힘" 토글(기본 켜짐) + 초 단위 입력(기본 10초)
  - 설정은 **팝업 인스턴스별로 독립적**이며 저장하지 않는다 — 다음에 새로 뜨는 팝업은 항상 기본값(켜짐/10초)으로 시작한다.
- **여러 팝업 동시 표시**: 서로 다른 카메라에서 동시에 탐지되면 각각 독립된 팝업으로 화면 한쪽(예: 우측 하단)에 스택 형태로 쌓아 보여준다. 개수 제한은 두지 않는다(카메라 대수가 적어 실무상 문제 없음).
- 자동 닫힘 타이머는 팝업이 열린 시점 기준으로 카운트하며, X를 누르면 즉시 닫히고 타이머는 무의미해진다.

## 알려진 제약 / 추후 조정 여지

- 8초 임계값은 초기 추정치다. 실제 CARLA DETR 틱 간격(~1.2초, 3틱마다 증거 생성 ≈ 3.6초)보다 넉넉히 크게 잡았지만, 실사용 후 너무 자주/드물게 재알림된다면 상수만 조정하면 된다.
- 팝업 설정 비저장은 의도된 단순화다 — 나중에 "항상 자동 닫힘 끄기"를 기억해달라는 요청이 오면 별도로 localStorage 저장을 추가하면 된다(지금은 범위 밖).

## 검증 방법

1. `npm run typecheck && npm run lint && npm run test` 통과 확인.
2. 수동 E2E: CARLA 카메라 연결 상태에서 실제로 사람이 탐지되면 팝업이 뜨는지, 10초 뒤 자동으로 닫히는지, X로 닫히는지, 톱니바퀴로 자동 닫힘을 끄면 안 닫히는지 확인.
3. 같은 카메라에서 사람이 계속 프레임에 머물 때 팝업이 반복해서 뜨지 않는지, 사라졌다가 8초 이상 지나 다시 나타나면 재알림되는지 확인.
4. 서로 다른 두 카메라에서 동시에 탐지시켜 팝업 두 개가 동시에 뜨는지 확인.
5. 우측 웹캠 DETR 테스트 패널에서 탐지시켜도 팝업이 뜨지 않는지 확인(카메라 id 재사용 케이스 포함).
6. EVENT TIMELINE 블록 호버 시 툴팁, 클릭 시 `ClipPlayer` 모달이 뜨는지, EVIDENCE CLIPS 섹션이 화면에서 사라졌는지 확인.
